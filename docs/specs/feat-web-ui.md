# feat-web-ui — Web Chat UI with Character Thoughts

## Goal

Add a browser-based chat UI for the AI Dungeon Master with a side panel showing character inner monologue, while keeping the existing terminal mode intact.

## Current Behavior

- Game runs exclusively via terminal (`run.py` → CLI loop)
- DM responses stream to stdout; user input via `input()`
- No character thought/inner monologue system exists

## Target Behavior

### Chat Panel (main area)
- Scrollable message history with player input and DM narrative
- Input box at bottom; sends on Enter
- DM tokens stream in real-time via WebSocket
- Dice rolls, NPC creation, damage/heal shown as distinct styled events
- TTFT displayed as a subtle indicator

### Character Panel (right sidebar)
- Card per character showing name, HP bar, role (PC/NPC)
- Each card has a scrolling thought log
- Thoughts generated by the LLM using a new `THINK:` command format
- Thoughts appear live as the DM response streams

### Architecture
- **Backend**: FastAPI + WebSocket at `/ws`
- **Frontend**: Single HTML file with vanilla JS + CSS (no framework)
- **Entry point**: `run_web.py` — separate from `run.py`
- **DM changes**: New `THINK:` command in system prompt + parser, new `get_response_events()` async generator

### WebSocket Protocol
Client sends:
```json
{"type": "action", "text": "Explore the cave"}
{"type": "new_game", "name": "Mitchell", "description": "A wandering bard"}
{"type": "load_game"}
{"type": "save"}
```

Server sends:
```json
{"type": "token", "text": "You "}
{"type": "command", "subtype": "roll", "text": "🎲 Mitchell dexterity..."}
{"type": "command", "subtype": "npc", "text": "✨ Created NPC: ..."}
{"type": "command", "subtype": "damage", "text": "⚔️ ..."}
{"type": "thought", "character": "Goblin", "text": "This human is distracted..."}
{"type": "state", "data": {characters: [...], location: "..."}}
{"type": "done"}
{"type": "error", "text": "..."}
```
